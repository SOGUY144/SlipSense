## WHAT (คืออะไร)

แอปพลิเคชันที่ให้เจ้าของร้านค้าและ SME **ถ่ายรูปสลิปโอนเงิน** แล้วให้ AI อ่าน จัดหมวดหมู่ และสรุปสุขภาพการเงินของร้านให้ทันที โดยไม่ต้องพิมพ์ข้อมูลใดๆ เอง พร้อมแสดง Dashboard รายรับ-รายจ่าย กำไรสุทธิ และคำแนะนำการประหยัดเงินที่วัดผลได้จริง

## WHY (ทำไมต้องมี)

ร้านค้าและ SME ในไทยกว่า 3.2 ล้านราย (90% ของธุรกิจทั้งหมด) ไม่รู้ว่าตัวเองกำไรหรือขาดทุนจริงๆ เพราะ

- สลิปโอนกองอยู่ในมือถือเป็นร้อยใบ ไม่มีเวลาอ่าน
- แอปบัญชีทั่วไปยุ่งยากเกินไป ต้องพิมพ์ข้อมูลเองทุกครั้ง
- จ้างนักบัญชีแพงเกินไปสำหรับร้านเล็กๆ
- ทำธุรกิจแบบ "เดาสุ่ม" ไม่มีข้อมูลช่วยตัดสินใจ

ผลคือเจ้าของร้านควบคุมต้นทุนหรือรู้ว่าควรตัดค่าใช้จ่ายอะไรไม่ได้เลย

## WHO (กลุ่มเป้าหมาย)

เจ้าของร้านโชห่วย แม่ค้าตลาด ร้านอาหาร และฟรีแลนซ์ที่รับโอนเงินเป็นหลัก มีรายรับ-รายจ่ายผ่านสลิปทุกวัน แต่ไม่มีระบบจัดการการเงินเลย รวมถึง SME ขนาดเล็กที่ยังไม่พร้อมลงทุนซื้อซอฟต์แวร์บัญชีราคาแพง

## HOW (ทำงานอย่างไร)

ใช้ Claude Vision API อ่านข้อมูลจากรูปสลิปโอนทุกธนาคาร (PromptPay, KBank, SCB, Krungthai ฯลฯ) ส่งให้ Claude API จัดหมวดหมู่อัตโนมัติว่าเป็นรายรับหรือรายจ่าย และหมวดอะไร เช่น ค่าวัตถุดิบ ค่าจ้าง ค่าเช่า รายได้ขาย จากนั้นแสดงผลเป็น Dashboard และคำแนะนำที่เข้าใจง่ายทันที

## ระบบหลักที่ต้องสร้าง

### 1. ระบบอ่านสลิป (OCR + AI Vision)
รับรูปสลิปจากทุกธนาคารในไทย ทั้งสลิปดิจิทัลและสลิปกระดาษที่ถ่ายรูป ดึงข้อมูลอัตโนมัติ: ชื่อผู้โอน/ผู้รับ จำนวนเงิน วันเวลา หมายเหตุ รองรับการอัพโหลดหลายใบพร้อมกัน

### 2. ระบบจัดหมวดหมู่อัตโนมัติ
AI วิเคราะห์บริบทแล้วจัดหมวดให้อัตโนมัติ แยกเป็นรายรับ (รายได้จากการขาย, รายได้อื่นๆ) และรายจ่าย (ค่าวัตถุดิบ, ค่าเช่า, ค่าจ้าง, ค่าน้ำค่าไฟ, ค่าขนส่ง, ค่าใช้จ่ายอื่นๆ)

### 3. หน้าตรวจสอบและแก้ไขข้อมูล
ก่อนบันทึกจริง ผู้ใช้ต้องตรวจสอบและแก้ไขข้อมูลที่ AI อ่านได้ (เผื่อ AI อ่านผิดหรือไม่มั่นใจ) มีระดับความมั่นใจ (high/medium/low) แสดงให้ผู้ใช้รู้ว่าควรตรวจทานข้อมูลไหนเป็นพิเศษ

### 4. Dashboard สุขภาพการเงิน
แสดงรายรับรวม รายจ่ายรวม กำไรสุทธิ ของเดือนนี้ เทียบกับเดือนที่แล้วเป็น % พร้อมรายการธุรกรรมล่าสุด

### 5. ระบบ AI แนะนำการประหยัด
เมื่อมีข้อมูลเพียงพอ AI จะวิเคราะห์และให้คำแนะนำที่เป็นตัวเลขจริง เช่น "ค่าวัตถุดิบสูงขึ้น 23% ถ้าเปลี่ยนซัพพลายเออร์ได้จะประหยัด ฿3,200/เดือน" ไม่ใช่คำแนะนำทั่วไปลอยๆ

### 6. หน้าวิเคราะห์ (Analytics)
กราฟแท่งแสดงรายรับ-รายจ่ายย้อนหลัง 6 เดือน และกราฟวงกลมแสดงสัดส่วนค่าใช้จ่ายแต่ละหมวด พร้อมรายละเอียดเป็น % และจำนวนเงิน

### 7. การเก็บข้อมูล
บันทึกข้อมูลธุรกรรมไว้ใช้ในครั้งต่อไป ไม่หายเมื่อปิดแอป มีปุ่มล้างข้อมูลสำหรับเริ่มใหม่

## Tech Stack

| ส่วน | เครื่องมือ | เหตุผล |
|---|---|---|
| OCR อ่านสลิป | Claude Vision API | อ่านสลิปภาษาไทยได้แม่น ทุกธนาคาร |
| วิเคราะห์ + จัดหมวด + คำแนะนำ | Claude API | จัดหมวดอัตโนมัติ วิเคราะห์ pattern การเงิน |
| Frontend | React | สร้างเร็ว responsive ใช้ได้ทั้ง mobile/desktop |
| กราฟ | Recharts | แสดงผลข้อมูลการเงินเป็นภาพที่เข้าใจง่าย |
| การเก็บข้อมูล | Storage API (key-value) | เก็บธุรกรรมไว้ใช้ข้ามครั้ง ไม่ต้องตั้ง server เอง |

## คำถามสำหรับลงพื้นที่ Field Research

### คำถามสำหรับอาจารย์ / Mentor
- AI อ่านสลิปที่มีรูปแบบหลากหลาย (ลายมือ, แสงไม่พอ, มุมเอียง) ความแม่นยำที่ยอมรับได้ในงาน hackathon ควรอยู่ที่กี่ %
- ถ้า AI อ่านสลิปผิด มีวิธีไหนที่ทำให้ผู้ใช้แก้ไขได้ง่ายโดยไม่เสียอรรถรส
- โมเดลธุรกิจแบบ Freemium เหมาะกับกลุ่ม SME ไทยไหม
- ข้อมูลการเงินของร้านค้าถือเป็นข้อมูลละเอียดอ่อน ควรระวังเรื่อง privacy แบบไหนเป็นพิเศษ
- ถ้าโจทย์ Surprise Condition เปลี่ยนกลางทาง ควรเตรียม pivot แบบไหนให้ดูมีไหวพริบ

### คำถามสำหรับร้านค้า
- ทุกวันนี้จัดการเรื่องรายรับ-รายจ่ายยังไงบ้าง
- รู้ไหมว่าเดือนที่แล้วกำไรหรือขาดทุนกี่บาท
- เคยลองใช้แอปบัญชีมาก่อนไหม ทำไมเลิกใช้หรือไม่เคยใช้
- อะไรคือเรื่องที่น่าเบื่อที่สุดในการจัดการเงินร้าน
- ถ้าแค่ถ่ายรูปสลิปแล้วระบบสรุปให้เลย คิดว่าจะใช้บ่อยแค่ไหน
- กังวลเรื่องข้อมูลการเงินรั่วไหลไหม
- ยินดีจ่ายเงินเพื่อใช้ฟีเจอร์แบบนี้ไหม ราคาเท่าไหร่ที่รู้สึกคุ้ม

## โครง Pitch Script (3 นาที)

1. **Hook (0:00–0:30)** — เปิดด้วยคำถามว่าเดือนนี้ร้านค้ากำไรเท่าไหร่ คนส่วนใหญ่ตอบไม่ได้
2. **Solution (0:30–1:00)** — เล่าว่าแค่ถ่ายรูปสลิป AI อ่านและสรุปให้ทันที
3. **Demo (1:00–1:45)** — โชว์ live demo ถ่ายรูปสลิปจริงและให้แอปสรุปทันที
4. **Validation (1:45–2:15)** — ใช้ข้อมูลจาก field research ยืนยันว่า pain point จริง
5. **Vision (2:15–2:45)** — บอกแผนต่อยอด เช่น เชื่อมระบบภาษี หรือ Credit Score
6. **Close (2:45–3:00)** — จบด้วยธีมงาน "Simple but Valuable"

## ไอเดียต่อยอดในอนาคต (ไม่อยู่ใน MVP ตอนนี้)

ระหว่างคิดไอเดียได้สำรวจแนวทาง **Gamification แบบสู้บอส** ไว้ด้วย เพื่อสร้างแรงจูงใจให้ผู้ใช้บันทึกข้อมูลสม่ำเสมอ แนวคิดคือเปลี่ยนปัญหาการเงินของร้าน (เช่น ค่าวัตถุดิบเกิน budget, กำไรติดลบ) ให้เป็น "บอส" ที่มี HP ลดลงเมื่อผู้ใช้แก้ปัญหาได้จริง พร้อมระบบภารกิจประจำวัน, Leaderboard ชุมชน, และ Badge สะสม

แนวคิดนี้ยังไม่ได้รวมเข้ากับ Prototype หลักเพื่อให้ MVP โฟกัสที่ประโยชน์ใช้งานจริงก่อน แต่เป็นแนวทางที่น่าพิจารณาเพิ่มในเวอร์ชันถัดไป หากต้องการเพิ่มความแตกต่างหรือ engagement ระยะยาว
