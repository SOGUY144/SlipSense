# 🎤 รวมคำถาม-คำตอบ (Q&A) เตรียมพร้อมสำหรับ Pitching นำเสนอผลงาน

ไฟล์นี้เตรียมไว้สำหรับใช้ฝึกซ้อมตอบคำถามกับกรรมการ ในการประกวดโครงงานหรือ Pitching แพลตฟอร์ม **SlipSense** ครับ

---

## 1. หมวด: ปัญหาและจุดเริ่มต้น (Pain Point & Origin)

**Q: ทำไมถึงเลือกทำโปรเจกต์นี้ ปัญหาที่แท้จริงคืออะไร?**
**A:** ปัญหาหลักคือ "SME และพ่อค้าแม่ค้าออนไลน์ส่วนใหญ่ ไม่รู้ว่าตัวเองกำไรหรือขาดทุนจริงๆ" ครับ เพราะการทำบัญชีเป็นเรื่องน่าปวดหัวและเสียเวลา หลายคนใช้วิธีเช็คแค่ยอดเงินคงเหลือในบัญชีธนาคาร ซึ่งไม่ได้สะท้อนถึงสุขภาพการเงินที่แท้จริง เราจึงสร้าง SlipSense ขึ้นมาเพื่อแก้ปัญหานี้ โดยลดภาระการจดบันทึกให้เหลือ 0 แค่ถ่ายรูปสลิป ทุกอย่างก็จะถูกจัดเก็บและคำนวณให้ทันทีครับ

**Q: ในตลาดก็มีแอปทำบัญชีเยอะแยะ (เช่น FlowAccount, PEAK, หรือแอปจดรายรับรายจ่ายทั่วไป) SlipSense ต่างจากแอปพวกนี้ยังไง?**
**A:** แอปเหล่านั้นดีมากครับ แต่ส่วนใหญ่ "ออกแบบมาให้นักบัญชีใช้" มีศัพท์เฉพาะเยอะ และปัญหาใหญ่ที่สุดของแอปเหล่านั้นคือ **"ผู้ใช้ยังต้องมานั่งพิมพ์ตัวเลขเอง"** ซึ่งทำให้คนส่วนใหญ่ขี้เกียจและเลิกใช้กลางทาง จุดเด่นของ SlipSense คือ "Zero-Friction" หรือการลดภาระผู้ใช้ให้ต่ำที่สุด แค่อัปโหลดสลิป AI จะดึงข้อมูล ยอดเงิน วันที่ และจัดหมวดหมู่ให้เองโดยอัตโนมัติ ผู้ใช้ไม่ต้องพิมพ์ตัวเลขเลยแม้แต่ตัวเดียวครับ

## 2. หมวด: เทคโนโลยีและ AI (Technology & AI)

**Q: AI ในระบบของคุณทำงานยังไง? ดึงข้อมูลจากสลิปได้ยังไง?**
**A:** เราใช้ **OpenAI (gpt-4o-mini)** ทำงานร่วมกับ **Vercel AI SDK** ครับ เมื่อผู้ใช้อัปโหลดสลิป ระบบจะส่งรูปภาพไปให้ AI Vision วิเคราะห์เพื่อดึงข้อมูล ยอดเงิน วันที่ และชื่อบัญชี จากนั้นเราใช้เทคนิค **Structured Outputs** เพื่อบังคับให้ AI คืนค่ากลับมาในรูปแบบ JSON ที่มีโครงสร้างชัดเจน ทำให้ระบบของเราสามารถนำข้อมูลไปบันทึกลง Database ได้อย่างแม่นยำและไม่ผิดพลาดครับ

**Q: ถ้า AI จัดหมวดหมู่ผิดล่ะ? ระบบจะทำยังไง?**
**A:** เราตระหนักถึงปัญหานี้ดีครับ เราจึงไม่ได้ให้ AI บันทึกข้อมูลลงฐานข้อมูลโดยตรง 100% แต่เรามี **ระบบให้ผู้ใช้ตรวจสอบ (Review)** ก่อนกดยืนยันเสมอ นอกจากนี้ล่าสุดเราได้พัฒนาระบบ **Shop Profile Context** ที่ให้เจ้าของร้านสามารถตั้ง "กฎเฉพาะ (Custom Rules)" ให้กับ AI ได้ เช่น "ถ้าร้านเราเป็นร้านเบเกอรี่ ซื้อไข่ไก่ ให้ตีเป็นค่าวัตถุดิบ 100%" ซึ่งกฎนี้จะถูกส่งไปเป็น Prompt ให้ AI ด้วย ทำให้ AI จัดหมวดหมู่ได้แม่นยำและเข้าใจบริบทของร้านนั้นๆ อย่างแท้จริงครับ

**Q: ทำไมไม่ใช้วิธีเชื่อมต่อ API กับธนาคารโดยตรงเลยล่ะ (Open Banking)?**
**A:** ปัจจุบันระบบ Open Banking API ในประเทศไทยยังไม่เปิดกว้างให้นักพัฒนารายย่อยหรือ Startup เข้าถึงได้อย่างอิสระครับ การสแกนข้อมูลจาก e-Slip หรือสลิปโอนเงิน จึงเป็นวิธีที่ทำได้จริง ตอบโจทย์ที่สุดในปัจจุบัน และครอบคลุมผู้ใช้งานทุกธนาคารครับ หากในอนาคตธนาคารเปิด API เราก็พร้อมที่จะเชื่อมต่อเข้ากับระบบที่มีอยู่แล้วได้ทันทีครับ

## 3. หมวด: โครงสร้างระบบ (Architecture & Stack)

**Q: คุณใช้ Tech Stack อะไรบ้าง และทำไมถึงเลือกใช้?**
**A:** เราใช้ Modern Stack ครับ:
- **Frontend / Framework:** Next.js (React) และ Tailwind CSS ช่วยให้พัฒนาได้เร็วและรองรับ SEO/SSR
- **Database:** Supabase (PostgreSQL) ร่วมกับ Drizzle ORM เลือกใช้เพราะเป็น Relational Database ที่เหมาะกับข้อมูลธุรกรรมการเงิน และมี Row Level Security (RLS) ที่ปลอดภัยมาก
- **AI Processing:** Vercel AI SDK เชื่อมกับ OpenAI ช่วยให้สลับโมเดล AI ในอนาคตได้ง่าย
- **Authentication:** ระบบ Login ด้วยเบอร์มือถือ (OTP) เพื่อลดความยุ่งยากในการจำรหัสผ่านของแม่ค้าครับ

**Q: แอปนี้ใช้งานได้จริงไหม หรือเป็นแค่ Mockup?**
**A:** ใช้งานได้จริง 100% ครับ ระบบฐานข้อมูล (Supabase) และ AI (OpenAI) ถูกต่อ API ไว้เรียบร้อยแล้ว ทุกสลิปที่สแกนในวันนี้เป็นการทำงานแบบ Real-time และบันทึกเข้าฐานข้อมูลจริง สามารถ Demo ให้ดูสดๆ ได้เลยครับ

## 4. หมวด: ความปลอดภัยของข้อมูล (Data Privacy & Security)

**Q: ข้อมูลสลิปและข้อมูลการเงินของร้านค้า เป็นข้อมูลที่ Sensitive มาก คุณจัดการเรื่องความปลอดภัยยังไง?**
**A:** ความปลอดภัยคือหัวใจหลักครับ เราจัดการ 3 ส่วน:
1. **ฐานข้อมูล (Supabase):** เราเปิดใช้ RLS (Row Level Security) ทำให้ผู้ใช้คนอื่นไม่สามารถเข้าถึงข้อมูลข้ามบัญชีกันได้โดยเด็ดขาด 
2. **AI Privacy:** รูปสลิปที่ส่งผ่าน API ไปยัง OpenAI จะไม่ถูกนำไปใช้ฝึก (Train) โมเดล AI ของพวกเขาตามนโยบาย Enterprise API
3. **การเข้ารหัส:** การรับส่งข้อมูลทั้งหมดผ่าน HTTPS เข้ารหัสปลอดภัยครับ

## 5. หมวด: โมเดลธุรกิจและอนาคต (Business Model & Future Scope)

**Q: แพลตฟอร์มนี้จะสร้างรายได้ยังไง? (Monetization)**
**A:** เราวางแผนเป็นระบบ **Freemium** ครับ
- **Basic (ฟรี):** สแกนสลิปได้จำกัดจำนวนต่อเดือน เหมาะสำหรับร้านค้ารายย่อยที่เพิ่งเริ่มต้น
- **Pro (Subscription รายเดือน/รายปี):** สแกนไม่จำกัด, สร้างหมวดหมู่ได้อิสระ, ตั้งกฎ AI ได้ละเอียด, และสามารถ Export ข้อมูลเป็น Excel/PDF เพื่อนำไปยื่นภาษี หรือส่งให้นักบัญชีต่อได้ครับ

**Q: แผนการพัฒนาต่อไปในอนาคต (Next Steps) คืออะไร?**
**A:** สเต็ปต่อไปของเราคือ:
1. **ระบบ Export สรุปภาษี:** สรุปยอดเพื่อช่วยคำนวณภาษีเงินได้ หรือภาษีหัก ณ ที่จ่าย
2. **ระบบ Credit Scoring:** นำข้อมูลรายรับ-รายจ่าย ไปสร้างเป็น "โปรไฟล์ความน่าเชื่อถือ" ให้ SME เพื่อใช้เป็นข้อมูลอ้างอิงในการขอสินเชื่อกับธนาคารในอนาคตครับ
3. **รองรับสแกนบิลกระดาษ:** นอกเหนือจาก e-Slip แล้ว AI ของเราจะถูกพัฒนาให้อ่านบิลเงินสดแบบกระดาษหรือใบเสร็จรับเงินที่เขียนด้วยลายมือได้ด้วยครับ
